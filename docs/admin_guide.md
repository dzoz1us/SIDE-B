# Руководство администратора
## Автоматизированная информационная система «SIDE B — Музыкальный магазин»

### 1. Системные требования
- Python 3.10+
- Node.js 18+
- PostgreSQL 15+

### 2. Установка и запуск

Порядок установки и настройки системы описан в разделе 3.2 пояснительной записки. Краткий порядок действий:

1. Клонировать репозиторий
2. Создать виртуальное окружение Python и установить зависимости
3. Создать базу данных `music_store` в PostgreSQL
4. Настроить параметры подключения в `config/settings.py`
5. Выполнить миграции: `python manage.py migrate`
6. Создать начальные данные: `python manage.py create_initial_data`
7. Установить зависимости фронтенда: `npm install`
8. Запустить сервер разработки: `python manage.py runserver` и `npm run dev`

После выполнения этих команд система доступна по адресу `http://localhost:5173`. Учётная запись администратора создаётся автоматически: `admin@vinyl.ru` / `admin123`. При первом входе рекомендуется сменить пароль.

### 3. Управление учётными записями менеджеров
1. Войдите под администратором
2. Перейдите в «Администрирование» → «Менеджеры» → «Добавить»
3. Заполните поля «Имя», «Фамилия», «Email», «Пароль»
4. Нажмите «Сохранить»

Для блокировки менеджера нажмите иконку щита рядом с его учётной записью. Заблокированный менеджер сможет войти в систему, но не получит доступа к панели управления. Для разблокировки нажмите иконку повторно.

### 4. Управление справочниками

В разделе «Администрирование» доступны:

- **Ансамбли** — создание, редактирование, удаление. В карточке ансамбля доступно управление составом: добавление и удаление музыкантов с указанием роли (исполнитель, дирижёр, руководитель) и инструмента
- **Музыканты** — создание, редактирование, удаление. Для каждого музыканта можно указать инструменты и загрузить фотографию
- **Произведения** — создание, редактирование, удаление. Произведение связывается с композитором из списка музыкантов
- **Филиалы** — создание, редактирование, удаление. Для каждого филиала указываются адрес, телефон и часы работы
- **Пластинки** — добавление, редактирование, удаление. Для каждой пластинки можно загрузить обложку и настроить треклист
- **Логи действий** — просмотр журнала операций

### 5. Резервное копирование

Для создания резервной копии базы данных выполните:
```bash
pg_dump -U postgres music_store > backup.sql

Для восстановления:
psql -U postgres music_store < backup.sql

### 6. Обновление системы

При выходе новой версии:

python manage.py migrate
npm install
